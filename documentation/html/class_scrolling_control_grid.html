ScrollingControlGrid Class Reference
[Utilities]

Overview

A grid of uicontrol objects.

ScrollingControlGrid is an alternative to Matlab's built-in uitable. Like uitable, it presents rows and columns of data and controls that you specify. ScrollingControlGrid has fewer explicit features than uitable and fewer arbitrary limitations.

The basic idea is that the ScrollingControlGrid contains a two-dimensional matrix of handles to uicontrol objects. It arranges all these uicontrols within a figure or uipanel, with the same row and column layout as the handles in the matrix.

The grid also provides a scroll bar so that the uicontrols can be moved up and down and large numbers of uicontrols can be accomodated.

The two-dimensional matrix of handles can contain redundant entries. uicontrols with multiple entries will have their Positions stretched to cover all of the individual row and column Positions. ScrollingControlGrid makes no attempt to prevent collisions among stretched uicontrols.

ScrollingControlGrid can manage uicontrol objects of any 'Style' and only specifies their 'Parent', 'Position', and 'Units' properties. So ScrollingControlGrid should accomodate many user interface tasks that call for a roughly grid-like presentation of data or controls.

Add a new uicontrol to the grid.

Parameters:
row the grid row to add the uicontrol to. row may be an array of row indices, in which case the uicontrol will be stretched vertically.
column the grid column to add the uicontrol to. column may be an array of column indices, in which case the uicontrol will be stretched horizontally.
varargin uicontrol property-value pairs. These are the same property-value pairs you might pass to the uicontrol() or set() function.

Returns a handle to a newly created uicontrol object, and adds the handle to the grid's two-dimensional matrix of handles, at row and column. The new uicontrol will use all property-value pairs contained in varargin. In addition, the new uicontrol will have its Parent property set by the grid.

row and/or column may be array of indices. In that case, the new uicontrol's handle will be added redundantly to multiple grid locations and it's appearance will be stretched to cover all the individual locations. Matlab's default indexing behavior is to add the handle to all combinations of row(i) and column(j) in the matrix.

The grid won't automatically update its appearance, after adding a new uicontrol. This allows controls to be added in batches and speeds up performance. To update the grid appearance, call repositionControls().

Remove the uicontrol at the given row and column.

Parameters:
row a scalar row index for the uicontrol to remove
column a scalar column index for the uicontrol to remove

Looks for a uicontrol handle in the controls matrix, at row and column. Calls delete() for any handle it finds there. Then sets all entries in the controls matrix that match the handle to 0. If removing a control results in empty rows on the bottom of the grid, or empty columns on the right, these will be trimmed.

repositionControls ( self   ) 

Refresh the layout of all uicontrols.

Recalculates the Position property for all uicontrol objects in the grid's controls matrix, to match the rows and columns of the matrix itself. Uses a constant height for all rows of controls, as specified by the grid's rowHeight property (character units). Stretches columns to fill the width of the entire grid.

When there are too many rows to fit vertically in the grid, they will run off the bottom and a slider bar will appear on the right side. It can be dragged to expose unseen controls.

After setting the individual uicontrol Positions, sets their Units property to normalized. Thus, subsequent resizing of the figure that contains the grid will cause the all the controls to stretch. If you call repositionControls() from the figure's ResizeFcn, the controls will maintain their constant height in character units.
